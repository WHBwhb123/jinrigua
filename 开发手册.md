# 「今日卦」项目开发手册（最终版）

> **项目名称**：今日卦
> **项目类型**：国学命理H5应用（手机端优先）
> **核心功能**：生辰八字卜卦 + 黄历查询 + 国学学习 + 社区广场 + 多种占卜方式
> **开发方式**：MVP优先，分阶段交付
> **目标平台**：手机浏览器 / 微信内置浏览器（响应式H5）
> **UI风格**：古朴古风（宣纸质感 + 金色点缀 + 书法字体）

---

## 一、你的任务

开发一个完整的「今日卦」H5应用，包含前端（Vue 3）和后端（Python FastAPI），用户通过密钥登录后可以使用全部功能。

## 五、核心算法实现要求

### 5.1 黄历数据
- 使用 `lunar-python` 获取公历转农历、天干地支、节气、节日
- 宜忌事项从预置数据或 `lunar-python` 的 `LunarToday` 获取
- 时辰吉凶：根据日干支和时辰计算

### 5.2 每日一卦（确定性映射）
```python
import hashlib
from datetime import date

def get_daily_hexagram(user_id, birth_date, today):
    """根据用户ID+生辰+日期生成固定的卦象"""
    key = f"{user_id}-{birth_date}-{today}"
    hash_val = hashlib.md5(key.encode()).hexdigest()
    hexagram_id = int(hash_val[:8], 16) % 64 + 1  # 1-64卦
    return hexagram_id
```

### 5.3 手动起卦（三枚铜钱法）
- 模拟三枚铜钱抛掷6次，得到6爻
- 使用 `ichingshifa` 库解析六爻
- 确定本卦和变卦

### 5.4 问题起卦
- 根据用户输入的问题文本哈希值 + 当前时间戳
- 生成随机数作为铜钱抛掷的种子
- 确保同一问题同一时间得到相同结果

### 5.5 文案匹配（三层降级）
1. **精确匹配**：本卦+变卦+吉凶 → 查 `gua_texts` 表
2. **部分匹配**：本卦+吉凶 → 随机选择一条
3. **默认文案**：兜底通用文案

### 5.6 星象占卜
- 预置当日星象数据（行星位置、星座、相位）
- 根据用户生辰星座和当日星象生成运势
- 可使用 `ephem` 库计算真实星象，或预置数据

### 5.7 塔罗牌占卜
- 预置78张塔罗牌数据（22张大阿卡纳 + 56张小阿卡纳）
- 根据用户问题和牌阵随机抽牌
- 解释正位/逆位含义

### 5.8 八字合婚
- 使用 `lunar-python` 计算双方八字
- 比较五行互补性、天干地支相合/相冲
- 给出匹配度评分和详细分析

### 5.9 姓名五格
- 根据姓名笔画计算五格（天格、人格、地格、外格、总格）
- 分析各格五行属性和运势
- 给出姓名评分和建议

### 5.10 择日择吉
- 根据用户生辰和事件类型
- 查询目标日期的黄历信息
- 分析是否适合进行该事件

## 十、关键业务规则

### 10.1 登录规则
- 用户输入邀请码直接登录，无需密码
- 邀请码由管理员后台生成，格式：`TGH-2026-XXXX`
- 首次登录后自动创建账号

### 10.2 生辰八字规则
- 支持公历/农历切换
- 填写出生日期 + 时辰 + 地点
- 系统自动计算八字（后端 `lunar-python`）
- 八字存入用户表，卜卦时自动带入
- 用户可随时修改生辰信息

### 10.3 卜卦次数规则
- 普通用户：每种模式每天1次
- VIP用户：每种模式每天3次
- 次数按自然日重置（0点）
- 使用 `divination_limits` 表记录

### 10.4 帖子审核规则
- 用户发帖后状态为 `pending`
- 管理员审核通过后状态变为 `approved`
- 审核不通过变为 `rejected`
- 只有 `approved` 的帖子在前端可见

### 10.5 学习进度规则
- 用户阅读文章时自动更新进度
- 滚动到底部触发完成状态
- 记录阅读时长

### 10.6 VIP订阅规则
- 订阅制（月付/年付）
- VIP到期后自动降级为普通用户
- VIP特权：卜卦次数增加 + 去广告 + 专属解读

## 十一、给Trae的特别说明

### 11.1 排盘部分不要自己写
- 八字计算、黄历数据、天干地支全部使用 `lunar-python`
- 六爻起卦使用 `ichingshifa`
- 不要自己实现这些算法

### 11.2 运势解读不要AI生成
- MVP阶段所有运势解读使用预置文案库（`gua_texts` 表）
- 文案由命理师提供，或使用模板批量生成
- 后期AI扩展功能（翻译/对话）再接入LLM

### 11.3 前端交互要求
- 底部导航栏固定，5个tab切换
- 卦象结果页要有动画（墨迹晕染、竹卷展开）
- 手动起卦要有铜钱抛掷动画
- 学习卡片要有翻转动画
- 所有页面要有过渡动画

### 11.4 后端安全要求
- 所有接口需要JWT认证
- 用户只能访问自己的数据
- 卜卦次数需要后端校验
- 邀请码需要后端验证

### 11.5 离线支持
- 使用 IndexedDB 存储卜卦历史
- 离线时可查看历史记录
- 在线时自动同步到云端
