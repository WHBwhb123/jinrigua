# 「今日卦」项目开发任务书

> **项目名称**：今日卦
> **项目类型**：国学命理H5应用（手机端优先）
> **核心功能**：生辰八字卜卦 + 黄历查询 + 国学学习 + 社区广场
> **开发方式**：MVP优先，分阶段交付
> **目标平台**：手机浏览器 / 微信内置浏览器（响应式H5）

---

## 一、你的任务

开发一个完整的「今日卦」H5应用，包含前端（Vue 3）和后端（Python FastAPI），用户通过密钥登录后可以使用全部功能。

## 三、数据库设计（7张表）

### 3.1 users - 用户表
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50),                           -- 昵称
    avatar VARCHAR(255),                            -- 头像URL
    birth_date DATE,                                -- 出生日期
    birth_hour VARCHAR(20),                         -- 出生时辰（如"子时"）
    eight_char VARCHAR(100),                        -- 生辰八字（缓存）
    invite_code VARCHAR(50) UNIQUE NOT NULL,        -- 邀请码/密钥（登录用）
    is_vip BOOLEAN DEFAULT FALSE,                   -- 是否VIP
    vip_expire_at TIMESTAMP,                        -- VIP到期时间
    created_at TIMESTAMP DEFAULT NOW(),
    last_login TIMESTAMP
);
```

### 3.2 divination_records - 卜卦记录表
```sql
CREATE TABLE divination_records (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),
    mode VARCHAR(20) NOT NULL,                      -- daily/manual/question
    gua_name VARCHAR(50),                           -- 本卦名称
    bian_name VARCHAR(50),                          -- 变卦名称
    ji_xiong VARCHAR(10),                           -- 吉/中/凶
    star INT,                                       -- 星级1-5
    result_json JSONB,                              -- 完整结果
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 3.3 posts - 帖子表
```sql
CREATE TABLE posts (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),
    tag VARCHAR(20) NOT NULL,                       -- 求测/解卦/交流/求助
    title VARCHAR(200),
    content TEXT NOT NULL,
    images JSONB,                                   -- 图片URL列表
    status VARCHAR(20) DEFAULT 'pending',           -- pending/approved/rejected
    like_count INT DEFAULT 0,
    comment_count INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 3.4 comments - 评论表
```sql
CREATE TABLE comments (
    id SERIAL PRIMARY KEY,
    post_id INT REFERENCES posts(id),
    user_id INT REFERENCES users(id),
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 3.5 likes - 点赞表
```sql
CREATE TABLE likes (
    id SERIAL PRIMARY KEY,
    post_id INT REFERENCES posts(id),
    user_id INT REFERENCES users(id),
```

### 3.6 learning_articles - 学习文章表
```sql
CREATE TABLE learning_articles (
```

### 3.7 learning_progress - 学习进度表

## 五、核心算法实现要求

### 5.1 每日一卦（确定性映射）

```python
import hashlib

def daily_gua_mapping(user_bazi: str, date_str: str) -> int:
    """
    同一个用户同一天 = 同一个卦象
    user_bazi: 如 "丙午年 乙未月 乙酉日 丙子时"
    date_str: 如 "2026-07-10"
    """
    raw = f"{user_bazi}_{date_str}"
    hash_val = hashlib.md5(raw.encode()).hexdigest()
    return int(hash_val[:8], 16) % 64  # 0-63 对应64卦
```

### 5.2 问题起卦

```python
def question_gua_mapping(question: str, date_str: str) -> int:
    """
    同一个问题同一天 = 同一个卦象
    """
    raw = f"{question}_{date_str}"
    hash_val = hashlib.md5(raw.encode()).hexdigest()
    return int(hash_val[:8], 16) % 64
```

### 5.3 文案匹配（三层降级）

```python
def match_gua_text(gua_name: str, bian_name: str, ji_xiong: str) -> dict:
    """
    从 gua_texts 表匹配解读文案
    优先级：精确匹配 > 本卦+吉凶 > 随机同吉凶
    """
    # 第一优先：精确匹配（本卦+变卦+吉凶）
    text = db.query(GuaText).filter_by(
        gua_name=gua_name, bian_name=bian_name, ji_xiong=ji_xiong, is_active=True
    ).first()
    
    if not text:
        # 第二优先：本卦名 + 吉凶
        text = db.query(GuaText).filter_by(
            gua_name=gua_name, ji_xiong=ji_xiong, is_active=True
        ).first()
    
    if not text:
        # 兜底：随机一条同吉凶等级的文案
        text = db.query(GuaText).filter_by(
            ji_xiong=ji_xiong, is_active=True
        ).order_by(func.random()).first()
    
    return text.to_dict() if text else default_fallback()
```

1. 根据64卦的吉凶规则，用AI批量生成白话解读
- 规则：本卦五行 + 变卦五行 + 日干 = 吉/中/凶
2. 生成后整理成SQL INSERT语句
3. 执行SQL导入数据库

**SQL示例**：

## 九、关键业务规则（必须严格遵守）

### 9.1 登录规则
- 只有邀请码能登录，没有账号密码
- 邀请码一次性使用（登录后标记为已使用）
- 登录成功后返回JWT，后续请求带 `Authorization: Bearer <token>`

### 9.2 生辰八字规则
- 用户必须填写 birth_date + birth_hour 才能卜卦
- birth_hour 必须是：子时(23-01)、丑时(01-03)、寅时(03-05)、卯时(05-07)、辰时(07-09)、巳时(09-11)、午时(11-13)、未时(13-15)、申时(15-17)、酉时(17-19)、戌时(19-21)、亥时(21-23)
- 前端做下拉选择，后端做格式校验

### 9.3 卜卦次数规则
- 普通用户：每种模式每天1次
- VIP用户：每种模式每天6次
- 判断逻辑：`WHERE user_id = ? AND mode = ? AND date(created_at) = today()`
- VIP判断：`user.is_vip == True AND user.vip_expire_at > now()`

### 9.4 帖子审核规则
- 所有新帖 status='pending'
- 只有 status='approved' 的帖子在广场可见
- 用户自己可以看到自己 pending 的帖子，别人看不到
- 帖子不能修改（简化版），只能删除重发

### 9.5 学习进度规则
- 用户查看文章后自动标记为 reading
- 阅读超过3分钟（或滚动到底部）标记为 completed
- 已读文章在列表中显示「已读」标识

#### 17.2.2 卜卦结果页「多风格解读」（新增功能，基于原有结果页增强）

**现状**：开发手册 4.3 卜卦响应包含固定字段，返回预置文案。

**融合方案**：在卜卦结果页增加「解读风格」切换，每种风格对应一套 prompt 模板，调用 LLM 生成差异化解读。

**注意**：手册 10.2 要求 MVP 运势解读不调用 AI。此项建议作为 **阶段三 P2 功能**，在预置文案基础上做增值服务，不替代原有逻辑。

**后端接口**：
```http
POST /api/divination/interpret
Authorization: Bearer ***

{
  "gua_name": "乾为天",
  "bian_name": "坤为地",
  "ji_xiong": "吉",
  "style": "scholar|fortune_teller|storyteller|modern"
}
```

**四种解读风格 prompt 模板**：

| 风格 | 角色设定 | 适用场景 |
|---|---|---|
| `scholar` | 古典文献学者，注疏式解读 | 学习型用户 |
| `fortune_teller` | 传统命理师，口语化吉凶 | 普通占卜用户 |
| `storyteller` | 说书人，故事化演绎 | 年轻用户/分享场景 |
| `modern` | 心理咨询师，现代隐喻解读 | 职场/情感决策场景 |

**实现要求**：
- 调用 LLM 时复用 16.2 节 prompt 结构
- 流式输出（SSE），前端逐字渲染
- 结果不存入数据库，仅本次会话展示
- 增加「换一解」按钮，切换风格重新生成

#### 17.3.4 手动起卦页「铜钱动效」实现

**现状**：开发手册 Manual.vue 为 6 次铜钱抛掷，未明确动效。

**融合实现方案**：

**方案 A（推荐，轻量）**：
- 纯 CSS 3D 铜钱
- 正面：`陽` 字（金色底）
- 背面：`陰` 字（墨色底）
- 抛掷动画：`rotateX` 从 0 到 `1800deg`，持续 1.5s
- 落地后根据随机数决定正反面

**方案 B（高保真，P2）**：
- 使用 Canvas 绘制铜钱纹理
- Three.js 做 3D 旋转（仅 high/fair 性能等级启用）
- 粒子特效模拟铜钱落地溅射

**代码结构**：
```vue
<template>
  <div class="coin-stage" @click="tossCoin">
    <div class="coin" :class="{ tossing: isTossing, heads: result === 'yang' }">
      <div class="coin-face front">陽</div>
      <div class="coin-face back">陰</div>
    </div>
  </div>
</template>

<style>
.coin {
  width: 80px; height: 80px;
  position: relative;
  transform-style: preserve-3d;
  transition: transform 1.5s cubic-bezier(0.25, 0.1, 0.25, 1);
}
.coin.tossing { transform: rotateX(1800deg); }
.coin-face {
  position: absolute; width: 100%; height: 100%;
  backface-visibility: hidden;
  border-radius: 50%;
  display: flex; align-items: center; justify-content: center;
  font-size: 28px; font-weight: bold;
}
.coin-face.front { background: linear-gradient(135deg, #C9A961, #E8D5A3); color: #2C2416; }
.coin-face.back { background: linear-gradient(135deg, #2C2416, #5C5346); color: #C9A961; transform: rotateX(180deg); }
.coin.heads .coin-face.back { transform: rotateX(0); }
.coin.heads .coin-face.front { transform: rotateX(180deg); }
</style>
```
